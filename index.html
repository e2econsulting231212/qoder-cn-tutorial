先分清 Qoder、QoderWork 和 CLI

视频里三个名称反复出现。它们面向的工作场景不同，先分清，后面就不会“下载错软件”或“在错误窗口里找按钮”。

概念官网入口视频 1
产品它是什么录屏中的用途你应在什么时候用

Qoder CN Desktop / IDE面向代码项目的桌面开发环境打开项目、查看文件树、Run Task、启动和排错需要阅读、修改、运行代码项目时
Qoder CN CLI终端中的 Qoder 命令行工具通过官网命令安装，或由 QoderWork 自动升级习惯终端或需要脚本化工作流时

访问国际站

浏览器输入 qoder.com。看到英文首页和右上角 Visit China Site →，说明当前是国际站。

点击 Visit China Site →，确认地址变为 qoder.com.cn。录屏后续注册、套餐、国内模型和 QoderWork CN 都从中国站进入。

03

注册账号，并让浏览器回到客户端

完整链路是“客户端发起 → 网站登录 → 浏览器授权打开 QoderWork CN → 回到工作台”。很多人卡在最后一步，不是账号失败，而是浏览器没有把登录状态交还给应用。

手机号阿里云账号视频 1–2

从网站或客户端点“登录/注册”

网站右上角点 登录，或在 QoderWork 欢迎页点 登录/注册。选择“个人登录”，再进入阿里云统一登录。

选择手机号或账号登录

没有账号时可用手机号验证码完成登录/注册；已有阿里云账号时可切换“账号登录”。按页面提示勾选用户协议和隐私政策。

完成浏览器登录

输入手机号、获取短信验证码并提交。验证码只应输入官方网站，不要粘贴到聊天窗口或截图中。

允许浏览器打开 QoderWork CN

出现“要打开 QoderWork CN 吗？”时，点击 打开 QoderWork CN。个人设备可勾选“始终允许”；公共电脑不建议勾选。

确认进入工作台

看到主标题“不止聊天，搞定一切”，左侧有“新任务 / 扩展 / 专家套件 / 技能 / 连接器”，即登录完成。

浏览器没有弹出“打开 QoderWork CN”怎么办？
  1. 回到 QoderWork 客户端重新点登录，让它再次生成回跳。
  2. 检查浏览器是否拦截了外部应用/协议跳转。
  3. 如果之前点过“取消”，刷新登录成功页并重新触发。
  4. 仍失败时完全退出 QoderWork，再从客户端重新开始登录，不要只停留在网站登录成功页。
隐私保护：原视频包含真实手机号、短信验证码、邮箱和账户 Credits。教程截图已使用脱敏版本；你在制作内部教程时也应同样处理。

遇到问题时，按根因排查

视频最有价值的部分不是“点了哪些按钮”，而是遇到环境问题后保留错误、检查真实状态，再执行最小修复。

不盲重试先检查后修复安全边界
现象最可能原因处理顺序成功标准
安装包打不开Mac 芯片架构选错，或 Windows 安装范围不合适确认 Intel / Apple Silicon；Windows 个人电脑先试 X64 User客户端可正常启动
网站登录了，客户端仍未登录浏览器未把状态回传给应用回客户端重发登录 → 允许“打开 QoderWork CN”进入“不止聊天，搞定一切”工作台
语音没有文字麦克风权限被拒绝或输入设备不对系统隐私设置开启权限 → 重启应用 → 再录一小句输入框出现可编辑转写文本
AI 读不到项目附加了错误子目录或压缩包未解压解压 → 选择含 package.json / README 的根目录 → 确认文件夹卡片AI 能列目录并读取启动配置
启动时报端口占用旧服务仍在运行检查端口 → 确认进程命令 → 只结束已确认旧实例 → 重启本地 URL 可访问
CLI 安装后仍是旧版本全局包未升级、PATH 指向旧位置查当前版本与路径 → 确认包名 → 升级 → 再次验证版本与可执行路径均符合预期
任务执行一半 Credits 不足余额不足或高倍率模型消耗较快打开“我的用量” → 查看两类 Credits → 调整模型或订阅任务可继续且余额变化可解释
项目运行了但浏览器没打开自动打开失败，但服务可能已启动让 Agent 报告实际 URL → 手动访问 → 检查服务日志页面可访问，而不是只看命令绿色状态

把错误交给 Qoder 的正确方式

诊断提示词

遇到错误时复制
这个任务遇到了问题。请先保留并读取完整错误信息，不要立刻重复执行同一命令。

请按以下顺序处理：
1. 说明失败发生在哪一步；
2. 检查当前系统、依赖、端口和相关进程；
3. 给出根因判断和证据；
4. 在执行可能终止进程、覆盖文件或修改环境的操作前先询问我；
5. 修复后用可见结果验证，而不只看命令返回成功。

文件安全

  • 先读后写
  • 限制在指定项目目录
  • 删除、覆盖、移动前确认
  • 敏感文件不上传、不截图

命令安全

  • 不照抄录屏 PID
  • 先识别进程再 kill
  • 保留完整错误输出
  • 安装后验证版本与路径

任务验证

  • “执行中”不等于完成
  • 待办变绿只是一个信号
  • 项目必须以页面可访问验证
  • 生成物应实际打开检查

成本控制

  • 大任务前看 Credits
  • 确认模型实时倍率
  • 避免重复发起同一任务
  • 并行 Agent 会增加工作量

四个可直接复用的提示词

整理工作并做周计划

只读扫描指定工作目录，按项目、进度、最近修改和待办整理，并生成下周一至周五计划。任何写入或不确定内容先询问我。

运行本地项目

先读 README 和 package.json 判断启动方式；检查依赖和端口后运行，并把实际本地 URL 告诉我和用浏览器打开。

诊断端口冲突

检查目标端口的占用进程和真实命令。确认它是本项目旧实例后再询问我是否结束；不要使用示例 PID。

安装 / 升级 CLI

先检查 qodercli 当前版本和路径，确认国内版包名及最新版本，再升级；完成后重新输出版本和路径并验证。

最终验收清单

账号与界面

  • 能从中国站进入正确产品
  • 客户端显示工作台而非欢迎页
  • 能打开“我的用量”
  • 能切换模型并看到倍率

任务与文件

  • 语音能转成正确文字
  • 右侧能看到任务拆解
  • 项目根目录已正确附加
  • AI 能读取 package.json

项目运行

  • 知道 dev 与 local 的差异
  • 端口冲突能先识别进程
  • 不照抄视频 PID
  • 浏览器能打开真实页面

CLI

  • 知道从官网复制系统命令
  • 能查 qodercli 版本
  • 知道国内版 npm 包名
  • 升级后再次验证路径与版本

术语速查

Credits

任务执行的资源额度。套餐内和资源包可能分别统计，并各有有效期。

倍率

模型菜单右侧的相对消耗系数；促销时可能很低，随时可能变化。

任务监控

右侧展示高层任务、待办、产物、默认工作目录以及技能与 MCP。

工作目录

智能体执行文件与命令操作的范围。选择错目录会让它读不到项目。

Run Task

Qoder IDE 右侧的 Agent 会话，用于理解项目、执行命令和排错。

可见验证

不只看“命令成功”，还要打开生成物、访问页面、检查版本或路径。
